# Markdown Notes (No LaTeX)

Use this as a quick visual tour of common Markdown features—everything here avoids LaTeX.

---

## Headings

# H1
## H2
### H3
#### H4
##### H5
###### H6

---

## Paragraphs & Line Breaks

A normal paragraph is just text wrapped at any width.

Line break (hard break) after two spaces at end of line:  
This sentence appears on the next line.

---

## Emphasis

*Italic*  
**Bold**  
***Bold + Italic***  
~~Strikethrough~~  

Escaped characters: \*not italic\*, \_not italic\_.

---

## Inline Code

Use backticks for `inline code` within a sentence.

---

## Code Blocks

```bash
# Shell
echo "Hello, Markdown"
```

```javascript
// JavaScript
function sum(a, b) {
  return a + b;
}
console.log(sum(2, 3));
```

```diff
# Diffs
- old line
+ new line
```

## Lists

### Unordered
- Item A
  - Sub-item A1
    - Sub-item A1a
- Item B

### Ordered
1. Step one
2. Step two
   1. Sub-step 2.1
   2. Sub-step 2.2
3. Step three

### Task Lists (GFM)
- [x] Write outline
- [x] Draft content
- [ ] Polish and publish

## Footnotes (GFM)

A sentence with a footnote reference.[^1] Another reference here.[^note]

[^1]: This is the first footnote.
[^note]: Another footnote with **formatting** and a link to <https://example.com>.

## Definition Lists (Parser-Dependent)

Term
: Definition for the term.

Another Term
: First definition
: Second definition

HTML comment (won’t render in many viewers):
